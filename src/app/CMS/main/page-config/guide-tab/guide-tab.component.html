Page Builder Guide

Learn how to create and configure dynamic pages with template binding.

How to Add Sections
@if (expandedSection() === 'sections') {
  1. Use Quick Presets — Click any preset button (Hero Banner, Content Text, Services Grid, etc.) in the "Quick Add" bar. This instantly adds a pre-configured section.
  2. Add Empty Section — Click "Add Section" to start from scratch with a blank section.
  3. Configure — Expand any section by clicking it. Set the template type, layout width, background, and custom CSS.
  4. Reorder — Use the up/down arrows to change section order on the page.
  5. Toggle Visibility — Use the switch to hide/show a section without deleting it.
}
Template Types & Templates
@if (expandedSection() === 'templates') {

Each section is rendered using a template from the Template Master.

  1. Template Type — Select a type (e.g., Hero, Content, CTA). This determines which templates are available.
  2. Template — Pick a specific template within that type. Each template has its own designed HTML layout.
  3. Dynamic Data — If a section has data configured (API or manual), the template can bind to it using variable syntax.
Templates are managed separately in the Template Master section of the CMS.
}
Dynamic Data Sources
@if (expandedSection() === 'data') {

Sections can pull data from external APIs or use manually entered values.

API Endpoint

  1. Set source to API Endpoint
  2. Enter the full API URL (e.g., https://api.example.com/services)
  3. Choose the HTTP method (GET, POST, PUT, DELETE)
  4. Add optional headers (e.g., Authorization tokens)
  5. Click Test Connection to verify and see available data keys
  6. Use the returned keys as {{key}} in your template

Manual Entry

  1. Set source to Manual Entry
  2. Click Add Field to create key-value pairs
  3. Choose a type for each field (text, number, image, URL, color, boolean, HTML)
  4. Enter the value
  5. Use the key name as {{key}} in your template
API responses are cached for 5 minutes by default. You can adjust the cache duration in the section settings.
}
Template Variable Syntax
@if (expandedSection() === 'binding') {

Use these patterns in your template HTML (in Template Master) to bind dynamic data:

Simple Variable
<h1>{title}}</h1>

Replaces with the value of the "title" field.

With Fallback
{title | Default Title}}

Uses "Default Title" if the field is empty or missing.

In Attributes
<img src="{heroImage}}" alt="{title}}">

Works inside HTML attributes too.

Array Loop
                  #each items 
                  <div class="card"> 
                  <h3>{{name}}</h3>
                  <p>{{description}}</p>
                  </div>
                  {{/each}}

Loops through an array. Each item's properties are available as variables.

Conditional
{{#if showBanner}}
                  <div class="banner">{{bannerText}}</div>
                  {{else}}
                  <div class="banner">Default Banner</div>
                  {{/if}}

Shows content only if the value is truthy.

}
GSAP Animations
@if (expandedSection() === 'gsap') {

Sections support GSAP scroll-triggered animations.

  1. Animation Key — Each section can have its own GSAP animation key, or inherit the page-level key.
  2. Animation Delay — Set a delay (in ms) for staggered reveal effects.
  3. Configure Animations — Actual animation definitions are managed in the GSAP Master section of the CMS.
GSAP animations are applied automatically when sections scroll into view. No manual setup needed per section.
}
